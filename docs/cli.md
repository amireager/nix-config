# ⚡ راهنمای ابزارهای خط فرمان و محیط‌های توسعه مرکزی (`CLI & DevShells Guide`)

در کانفیگ سیستم شما، ابزارهای خط فرمان به **دو لایه کاملاً مجزا و بهینه** تقسیم شده‌اند:
1. **لایه اول (ابزارهای دائمی و روزمره):** ابزارهایی بسیار سبک و سریع که همیشه در ترمینال‌های شما آماده کار هستند.
2. **لایه دوم (ابزارهای تحلیلی، بازرسی سنگین و محیط‌های توسعه مرکزی - `devShells`):** ابزارهایی که با دستور جادویی **`dev <environment>`** فقط و فقط هنگام نیاز به صورت ایزوله لود می‌شوند.

---

## 🚀 بخش اول: ابزارهای دائمی و روزمره شل (`Global Everyday CLI`)

این ابزارها در `modules/home/cli/tools.nix` نصب شده و در هر ثانیه در دسترس هستند. مثال‌های کاربردی زیر نحوه استفاده از پرکاربردترین آن‌ها را نشان می‌دهد:

### ۱. جایگزین‌های مدرن برای دستورات کلاسیک لینوکس
* **`eza` (جایگزین فوق‌مدرن `ls` مجهز به آیکون و گیت):**
  * نمایش لیست فایل‌ها همراه با وضعیت گیت و آیکون: `eza --icons --git -l`
  * نمایش ساختار درختی ۲ سطحی دایرکتوری جاری: `eza --tree --level=2 --icons`
* **`bat` (جایگزین رنگی و شماره‌دار `cat`):**
  * نمایش محتوای یک اسکریپت با هایلایت سینتکس و شماره خط: `bat script.py`
  * نمایش متون بدون شماره خط (ساده و تمیز برای پایپ کردن): `bat --style=plain file.json`
* **`ripgrep` (`rg` - سریع‌ترین جستجوکننده متن در جهان):**
  * جستجوی کلمه `TODO` در تمام فایل‌های پایتون پروژه: `rg "TODO" -t py`
  * جستجوی بدون حساسیت به حروف کوچک/بزرگ: `rg -i "config"`
* **`fd` (جایگزین سریع و کاربرپسند `find`):**
  * پیدا کردن تمام فایل‌های با پسوند `.kdl` در سیستم: `fd -e kdl`
  * پیدا کردن دایرکتوری‌هایی با نام `secrets`: `fd -t d "secrets"`
* **`sd` (جایگزین 직هودی و مدرن `sed` برای جستجو و جایگزینی):**
  * جایگزینی تمام کلمات `old_func` با `new_func` در فایل جاری: `sd "old_func" "new_func" file.py`
  * جایگزینی دسته‌جمعی در تمام فایل‌های پروژه: `fd -t f | xargs sd "http://localhost" "https://api.site.com"`

### ۲. ابزارهای مدیریت فایل، شبکه و مانیتورینگ
* **`xh` (کلاینت HTTP تمیز و سریع - جایگزین مدرن `curl` و `Postman`):**
  * ارسال درخواست GET به یک API: `xh https://api.github.com/users/amireager`
  * ارسال داده JSON با متد POST: `xh POST https://httpbin.org/post name=Amir role=developer`
* **`doggo` (کلاینت مدرن DNS - جایگزین رنگی `dig`):**
  * استعلام رکوردهای A و MX یک دامنه: `doggo google.com A MX`
  * استعلام از طریق سرور خاص (مثلاً کلودفلر): `doggo google.com @1.1.1.1`
* **`mtr` & `gping` (ابزارهای پینگ و تریس‌روت گرافیکی در ترمینال):**
  * پینگ گرفتن زنده با نمودار خطی از ۳ سرور هم‌زمان: `gping google.com 8.8.8.8 1.1.1.1`
  * اجرای تریس‌روت همراه با آمار اتلاف بسته: `mtr -t google.com`
* **`just` (اجراکننده دستورات پروژه - جایگزین ساده `Makefile`):**
  * اگر فایلی به نام `justfile` در پروژه باشد، با دستور `just --list` تمام تسک‌ها نمایش داده شده و با `just build` یا `just deploy` اجرا می‌شوند.
* **`glow` (رندرکننده زیبای اسناد Markdown در ترمینال):**
  * مطالعه فایل‌های مستندات به صورت استایل‌دار: `glow docs/cli.md`

## 📦 بخش دوم: مرکز فرماندهی محیط‌های پویا و ایزوله (دستور `dev`)

برای جلوگیری از سنگین شدن سیستم و شلوغی `PATH`، تمام زنجیره‌های کامپایل، ابزارهای مهندسی داده و موتورهای سندباکس در پوشه `shells/` پروژه قرار دارند.

### 💡 نحوه کارکرد دستور `dev`
دستور `dev` یک اسکریپت هوشمند سیستمی است که در هر کجای کامپیوتر وارد کنید، بررسی می‌کند که آیا محیط مدنظر قبلاً در کش محلی ریشه‌سازی شده یا نه؛ اگر نبود دانلود و ریشه‌سازی می‌کند و در کمتر از چند میلی‌ثانیه شما را به ترمینال تخصصی می‌برد:

```bash
# نمایش لیست تمام محیط‌های موجود در سیستم همراه با توضیحات
dev

# ورود تعاملی به یک محیط تخصصی (مثلاً پایتون یا راست)
dev python
dev rust

# اجرای مستقیم یک دستور درون محیط تخصصی (بدون ورود تعاملی)
dev cli hyperfine --warmup 3 'python3 script.py' 'pypy3 script.py'
```

## ⚡ بخش سوم: راهنمای ابزارهای تحلیلی و بازرسی سنگین (`dev cli`)

با ورود به محیط `dev cli`، قدرتمندترین ابزارهای آنالیز و دیباگ لینوکس در اختیار شماست:

### ۱. جستجوی ساختاری بر اساس درخت نحو (`ast-grep` / `sg`)
ابزار `ast-grep` برخلاف `ripgrep` که متن خام را جستجو می‌کند، **درخت نحو انتزاعی (AST)** کد را می‌فهمد! بنابراین پرانتزها، کامنت‌ها و قالب‌بندی‌ها تاثیری در دقت آن ندارند.
* **مثال جستجو:** پیدا کردن تمام فراخوانی‌های تابع `print(...)` در کد پایتون:
  ```bash
  sg -p "print($A)" -l py
  ```
* **مثال رفکتور خودکار:** تبدیل تمام `print(x)` ها به `logger.info(x)` در پروژه:
  ```bash
  sg -p "print($A)" -r "logger.info($A)" -l py --update-all
  ```

### ۲. بنچمارک‌گیری بسیار دقیق از دستورات (`hyperfine`)
برای مقایسه سرعت اجرای دو اسکریپت یا دو ابزار مختلف با دقت میلی‌ثانیه و رسم آمار انحراف معیار:
```bash
hyperfine --warmup 3 --runs 10 'fd "main" .' 'find . -name "main"'
```

### ۳. مانیتورینگ پیشرفته پردازش‌ها و پهنای باند (`procs` & `bandwhich`)
* **`procs` (جایگزین درختی و رنگی `ps`):**
  * جستجوی پردازش‌های مربوط به نئوویم یا فایرفاکس: `procs nvim firefox`
  * نمایش درختی با فیلتر مصرف رم بالا: `procs --tree --sortd mem`
* **`bandwhich` (رصد زنده پهنای باند شبکه به تفکیک پردازش و اتصال):**
  * اجرای مانیتورینگ زنده (نیازمند دسترسی ریشه یا قابلیت شبکه): `sudo bandwhich`

### ۴. پردازش و آنالیز داده‌های ساختاریافته (`dasel`, `fx`, `jc`, `jless`)
* **`dasel` (جستجو و ویرایش چندمنظوره JSON, YAML, TOML, XML):**
  * استخراج نسخه یک پکیج از فایل `pyproject.toml` یا `Cargo.toml`:
    `dasel -f pyproject.toml 'tool.poetry.dependencies.python'`
* **`fx` (اکسپلورر تعاملی JSON):**
  * باز کردن و پیمایش درختی یک فایل حجیم JSON: `fx data.json`
* **`jc` (تبدیل خروجی متنی دستورات لینوکس به JSON استاندارد):**
  * تبدیل خروجی دستور `ifconfig` یا `ls` به JSON جهت پردازش با `jq`:
    `ifconfig | jc --ifconfig | jq '.[] | .name, .ipv4_addr'`

### ۵. تولید خودکار عبارت باقاعده (`grex`)
اگر نوشتن Regex برایتان سخت است، چند نمونه متن به `grex` بدهید تا پترن دقیق آن را برایتان بسازد:
```bash
grex "2026-07-20" "2025-01-15" "2024-12-01"
# خروجی: ^202[4-6]-(?:01|07|12)-(?:01|15|20)$
```

### ۶. مدیریت صف کارهای پس‌زمینه طولانی (`pueue`)
وقتی می‌خواهید چند بیلد سنگین یا دانلود طولانی را در پس‌زمینه اجرا کنید بدون اینکه ترمینال قفل شود:
```bash
# اضافه کردن کارها به صف پس‌زمینه
pueue add -- cargo build --release
pueue add -- yt-dlp https://youtube.com/playlist...

# مشاهده وضعیت و درصد پیشرفت صف
pueue status

# مشاهده لاگ زنده یک کار خاص
pueue log 0
```

## 🛡️ بخش چهارم: راهنمای شل امنیتی و جعبه‌شنی (`dev sec`)

محیط `dev sec` بر پایه موتور قدرتمند **`Bubblewrap (bwrap)`** مهندسی شده و دارای ۴ اسکریپت دفاعی برای تست باینری‌ها و اسکریپت‌های ناشناخته است:

### ۱. محیط تعاملی محدود به پوشه جاری (`dir-sandbox`)
با اجرای دستور `dir-sandbox` درون این شل، یک ترمینال جدید باز می‌شود که در آن کل فایل‌های سیستم‌عامل (`/usr`, `/etc`, `/nix`) به صورت **فقط‌خواندنی (Read-Only)** هستند، اینترنت کاملاً قطع است (`--unshare-net`)، و پوشه خانه شما (`/home/amir`) **منحصراً به همان دایرکتوری جاری که در آن حضور دارید محدود شده است!** هیچ برنامه‌ای نمی‌تواند از این پوشه خارج شود یا به کلیدهای SSH دسترسی پیدا کند.

### ۲. اجرای آفلاین باینری ناشناخته (`safe-run`)
برای تست فوری یک فایل اجرایی ناشناخته بدون اینترنت و با دسترسی خواندن/نوشتن محدود به پوشه جاری:
```bash
safe-run ./unknown-binary --test-flag
# یا اگر بدون آرگومان بزنید، وارد شل تعاملی آفلاین می‌شود
safe-run
```

### ۳. اجرای آنلاین باینری ناشناخته (`safe-net-run`)
برای اجرای اسکریپت‌های نصب‌کننده آنلاین (`curl | sh`) یا ابزارهای اینترنتی که نیاز به دانلود دارند اما نباید به اسناد شخصی یا کلیدهای امنیتی شما دسترسی داشته باشند:
```bash
safe-net-run ./online-installer.sh
```

### ۴. مسدودکننده اینترنت برای برنامه‌های عادی (`net-block`)
اگر می‌خواهید یک برنامه معمولی (که دسترسی کامل به فایل‌های عادی نیاز دارد) را اجرا کنید اما **کارت شبکه و اینترنت آن به طور کامل قطع شود**:
```bash
net-block ./my-local-tool --work-offline
```

## 📦 خلاصه شل‌های باقیمانده در `dev`

| دستور فراخوانی | ابزارهای موجود و کاربرد اصلی | اتوماسیون و راهنما |
| :--- | :--- | :--- |
| `dev python` | `python3`, `poetry`, `ruff`, `pyright`, `ipython`, `virtualenv` | سوال تعاملی `y/N` برای ساخت `.venv` قبل از ورود. |
| `dev rust` | `cargo`, `rustc`, `rust-analyzer`, `clippy`, `rustfmt`, `pkg-config`, `openssl` | اتصال خودکار نئوویم به LSP از طریق `direnv-vim`. |
| `dev go` | `go`, `gopls`, `golangci-lint`, `delve` | اتصال خودکار نئوویم به LSP و ابزار دیباگ. |
| `dev build` (`dev c`) | `gcc`, `clang`, `cmake`, `make`, `ninja`, `pkg-config`, `openssl`, `cargo` | محیط کامل کامپایل پروژه‌های C/C++/Rust کلون‌شده از گیت‌هاب. |
| `dev nix` | `statix`, `deadnix`, `alejandra`, `nixd`, `nix-tree`, `nvd`, `nix-check` | بازرسی سلامت سینتکس و ارزیابی Flake. |
| `dev py-rust` | ترکیب هم‌زمان تمام پکیج‌های پایتون و راست در یک ترمینال واحد | مناسب پروژه‌های چندزبانه. |
