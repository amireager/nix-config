# ⚙️ راهنمای معماری سیستمی، دیسک NVMe، رم ZRAM و شبکه (`NixOS System Architecture & Kernel Tuning`)

در کانفیگ هسته سیستمی شما (`modules/nixos/core.nix`، `network.nix` و `hardware.nix`)، مجموعه‌ای از بهینه‌سازی‌های عمیق در سطح کرنل لینوکس اعمال شده است تا حداکثر کارایی، سرعت پاسخ‌دهی و طول عمر سخت‌افزار تضمین شود.

---

## 💾 ۱. بهینه‌سازی دیسک‌های پرسرعت (`NVMe / SSD I/O Tuning`)

دیسک‌های SSD و NVMe دارای طول عمر مشخصی بر اساس حجم نوشتن (`TBW`) هستند. برای جلوگیری از استهلاک بی‌جهت و افزایش سرعت خواندن:

### الف) حذف رکورد زمانی دسترسی (`noatime` & `nodiratime`)
در فایل `hardware.nix`، پارتیشن ریشه با گزینه‌های زیر مونت شده است:
```nix
fileSystems."/".options = [ "noatime" "nodiratime" ];
```
* **تأثیر واقعی:** در حالت پیش‌فرض، لینوکس با هر بار خواندن هر فایل (مثلاً اسکن فایل‌ها توسط گیت یا کامپایلر)، یک رکورد زمانی (`atime`) روی هارد می‌نویسد! با حذف این قابلیت، میلیون‌ها نوشتن ریز و اضافی از روی SSD برداشته شده و جستجوی فایل‌ها سریع‌تر می‌شود.

### ب) نگهداری و پاکسازی دوره‌ای دیسک (`services.fstrim`)
سرویس `fstrim` به صورت هفتگی (`interval = "weekly"`) بلوک‌های حذف‌شده دیسک را به کنترلر NVMe اعلام می‌کند تا سرعت دیسک در طول سال‌ها افت نکند.

## 🏎️ ۲. روان بودن دسکتاپ زیر بار سنگین (`Kernel Dirty Buffer Tuning`)

یکی از مشکلات رایج لینوکس این است که هنگام کپی فایل‌های چند گیگابایتی یا بیلد سنگین Nix، موس و دسکتاپ دچار لگ یا فریز ناگهانی می‌شوند. برای ریشه‌کنی این مشکل، تنظیمات زیر به کرنل (`boot.kernel.sysctl`) تزریق شده است:

```nix
"vm.dirty_background_ratio" = 5;
"vm.dirty_ratio" = 10;
```
* **تأثیر واقعی:** به جای اینکه کرنل اجازه دهد ۲۰ تا ۴۰ درصد رم پر از داده‌های کثیف (Dirty Pages) شود و بعد همه را یکباره روی هارد تخلیه کند (که باعث قفل شدن سیستم می‌شود)، اکنون به محض اینکه بافر به ۵٪ رسید، به صورت روان، پیوسته و بدون تاخیر روی دیسک نوشته می‌شود.

---

## 🧠 ۳. معماری حافظه و رم فشرده (`ZRAM + Memory Watermarks`)

برای اینکه سیستم شما حتی با ده‌ها تب مرورگر و بیلد‌های سنگین هرگز دچار کمبود رم (`OOM`) یا کندی Swap روی هارد دیسک نشود، مکانیزم **`ZRAM`** فعال شده است:

```nix
zramSwap = {
  enable = true;
  memoryPercent = 100; # اختصاص ۱۰۰٪ رم به عنوان بلاک فشرده‌ساز zstd
};
boot.kernel.sysctl = {
  "vm.swappiness" = 180;
  "vm.page-cluster" = 0;
  "vm.watermark_scale_factor" = 125;
};
```
* **چرا `page-cluster = 0` حیاتی است؟** در Swap سنتی روی هارد دیسک، لینوکس برای کاهش تاخیر چرخش دیسک، صفحات را به صورت دسته‌ای (Cluster) می‌خواند. اما چون ZRAM مستقیماً درون حافظه رم است و سرعت نور دارد، تنظیم `page-cluster = 0` باعث می‌شود صفحات تکی و در ۰ میلی‌ثانیه فشرده و بازگشایی شوند.

## 🌐 ۴. شبکه بهینه BBR و حریم خصوصی DNS (`TCP & dnscrypt-proxy`)

### الف) الگوریتم کنترل ازدحام BBR (`network.nix`)
تنظیمات شبکه روی الگوریتم مدرن گوگل (`TCP BBR`) و صف `fq` تنظیم شده است:
```nix
"net.ipv4.tcp_congestion_control" = "bbr";
"net.core.default_qdisc" = "fq";
"net.ipv4.tcp_fastopen" = 3;
```
این الگوریتم سرعت پینگ و پهنای باند را در اینترنت پرنوسان به شدت بهبود می‌بخشد.

### ب) امنیت و رمزنگاری DNS
سرویس `dnscrypt-proxy` روی پورت ۵۳ محلی (`127.0.0.1`) فعال است و تمام درخواست‌های DNS را قبل از خروج از لپ‌تاپ رمزنگاری می‌کند تا از شنود و نشت درخواست‌ها (`DNS Leak`) جلوگیری شود.

## ⚡ ۵. معماری بوت سریع و بی‌صدا (`Fast & Silent Boot`)

برای کاهش زمان استارت‌آپ سیستم به چند ثانیه:
* **`initrd.systemd.enable = true`:** ماژول‌ها و درایورها در مرحله اولیه بوت توسط `systemd` به صورت موازی (Async) لود می‌شوند.
* **`consoleLogLevel = 3` و پارامترهای `quiet / splash`:** متون طولانی و شلوغ در بوت حذف شده و فقط ارورهای حیاتی نمایش داده می‌شوند.
* **`loader.timeout = 1`:** تاخیر منوی بوت‌لودر به ۱ ثانیه کاهش یافته است.

---

## 🗄️ ۶. اتوماسیون پاکسازی هارد و لاگ‌ها (`Journald & Store Deduplication`)

* **کنترل لاگ‌های سیستمی (`SystemMaxUse=250M`):** حجم لاگ‌های `journald` هرگز از ۲۵۰ مگابایت تجاوز نمی‌کند.
* **یکپارچه‌سازی خودکار استور (`nix.optimise.automatic = true`):** هر هفته اجرا شده و فایل‌های هدر یا کتابخانه‌های تکراری در `/nix/store` را به `Hardlink` تبدیل می‌کند که به طور خودکار گیگابایت‌ها فضای هارد را آزاد نگه می‌دارد.
